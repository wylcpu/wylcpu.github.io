DOM之增删改查

發表於 2015-06-26   |   分類於 DOM

今天看了一下，JavaScript权威指南的DOM操作部分，就记录一下。

Dom

文档对象模型（DOM）是表示和操作HTML和XML文档内容的基础API。

获取文档元素

  • getElementById()
  • getElementsByTagName()
  • getElemtnsByName()
  • getElmeentsByClassName()
    后面这两个是获取的NodeList对象，是个只读类数组对象
    ps.在怪异模式下，class属性和css样式表是不区分大小写的
  • 还有一些快捷方式获取form,a,img元素的方法
    • document.forms
    • document.links
    • document.images
      这些获取的是一个HTMLCollectioin对象，也是一个只读的类数组对象
      PS.Nodelist And HTMLCollection对象都是 实时 更新的，文档中的Dom变得话，他们包含的元素也是会立即改变的，
      可以用下标的方式获取，可以通过定义元素的name获取指定的标签，也可以通过数字顺序取得元素
  • 还有特殊的单个元素，html,body,head
    • document.documentElement
    • document.body
    • document.head
      其中document的documentElement属性指的是文档的根元素，在HTML中指的是<HTML>元素
  • querySelectorAll()
    接受一个css选择器的字符串作为参数，返回匹配元素的NodeList对象，注意这个NodeList不是实时更新的
  • querySelector()
    这个方法和上面的是一样的，只是他知识获取匹配的第一个元素
    注意：在元素上调用时，指定的选择器仍然在整个文档中进行匹配，然后过滤出结果集以便它只包含指定元素的后代元素；这很反常，但这是为了你可以在选择器字符串中可以包含元素的祖先，而不仅仅选择器中只能包含后代元素的选择器

文档的遍历

下面有两套API

节点树文档解析API

就是节点树API，也就是node对象中API去解析文档

node对象

属性

  • parentNode
    获取一个节点的父节点,这个父节点只能是Element类型
  • childNodes
    获取子节点，nodeList对象是实时更新的
  • firstChild,lastChild
    获取节点的第一个子节点和最后一个子节点
  • nextSibling,previoursSibling
    ps.上面的这些API获取的节点都是包括 文本节点，注释节点，元素节点；获取不到就返回null
  • nodeType
    获取节点的类型，常见的类型有9代表document节点，1代表Element节点，3代表Text节点，8代表Comment节点，11代表DocumentFragment节点
  • nodeValue
    获取 Text节点 或者是 comment节点 的文本内容
  • nodeName
    获取的是元素的标签名,如果是Element节点，则返回的标签名都是大写字母，如果是文本节点则返回#text；如果是注释节点则返回#comment

元素树文档解析API

这里解析就忽略包括文本（text）节点和注释（comment）节点；我们一般是使用这个的，上面的API会连元素之间的空格什么的都解析，我们大部分的时候是只关心Element的；所以这套API我们更需要，但是他们没有标准化，但是浏览器厂家都支持

  • children
    也是一个nodeList对象，并且它只获取Element元素
  • firstElementChild,lastElementChild
  • nextElementSibling,previousElementSibling
  • childElementCount
    获取子元素的数量

文档属性

获取或设置元素的属性
  • getAttribute(),setAttribute()
    用于获取和设置属性的
    ps.html属性名不区分大小写，JavaScript属性名区分大小写
  • hasAttribute(),removeAttribute()
    用于检查是否有这个属性，和移除这个属性
  • attributes
    这个是node类型定义的attribute属性，并且这个属性是只读的
    非Element对象的节点，该属性为null

dataset属性

这个一个HTML5在Element对象上定义的一个数据集属性，HTML元素中以data-为前缀
注意，在设置和获取属性的时候，不需要带data前缀
ps，也可用getAttribute,和setAttribute来设置，但是要要带前缀data

元素内容

获取或设置HTML内容
  • innerHTML
    用于设置或读取元素的的内容，在设置的时候会调用web浏览器的解析器
  • outerHTML
    用于设置或读取 元素本身 及其内容
  • insertAdjacentHTML
    HTML5中标准化，它可以将HTML标记字符串插入到指定的元素 相邻 位置
    第一个参数是一个设置插入位置的字符串，第二个参数是要插入的内容
    它有四个值
    • beforebegin
    • afterbegin
    • beforeend
    • afterbegin
纯文本内容

不对内容中HTML标记中使用的尖括号和&进行转义，就只是当作普通的文本内容

  • textContent,innerText
    textContent可以获取 后代 所有元素的文本内容 ，IE9+支持
    innerText,火狐中不支持这个

创建、插入、删除

创建节点
  • createElement()
    创建一个新的Element节点
  • createTextNode()
    创建一个文本（text）节点
  • cloneNode()
    可以传入一个参数，false代表浅复制(就复制元素本身，不复制后代元素)，true代表深复制（复制元素本身及后代元素）
插入节点
  • appendChild()
    插入在指定的节点上，使指成为那个节点的最后一个子节点。
  • insertBefore()
    插入在一个节点的前面
    它接收两个参数，第一个参数是：待插入的节点，第二个参数是已存在的节点，新节点将插入在它的前面，调用要在它父元素上调用。如果第二个参数传入null，就和appendChild的行为是一样的
    ps.上面两个方法如果是将已经存在与文档中再次插入时，它会自动删除当前节点并在新的位置重新插入该节点。
删除和替换节点
  • removeChild()
    在父元素上调用该方法，并把需要删除的子节点作为参数传递给他
    oEle.parentNode.removeChild(oEle),这样就不需要知道父元素是谁了
  • replaceChild()
    替换一个节点
    在父元素上调用，接受两个参数，第一个参数是新节点，第二个是要替换的节点

DocumentFragment

  • createDocumentFragment()
    它的parentNode是null，将文档片段插入到指定的节点位置，是将文档片段的所有子节点插入到文档中，而非文档片段本身

  • 基本constructor（构造器）

代码：

function Car(model,year,miles) {
    this.model = model;
    this.year = year;
    this.miles = miles;
    this.toString = function() {
        return this.model + ' has done ' + this.miles + ' miles';
    }
}
var car= new Car('红旗',10,20000);
console.log(car.toString());

缺点是，这个构造器实例化的对象的函数方法不能公用，每个新对象都有一个自己的方法

  • 带原型的constructor(构造器)

把上面的代码改为

function Car(model,year,miles) {
    this.model = model;
    this.year = year;
    this.miles = miles;
}
Car.prototype.toString = function() {
    return this.model + ' has done ' + this.miles + ' miles';
}
var car1 = new Car('benz',5,30000);
var car2 = new Car('bmw',6,50000);
console.log(car1.toString());
console.log(car2.toString());

这次的代码，car1,car2是用的一个toString方法，实现了所有实例共享一个方法

代码：

var myModule = {
    //基本属性
    myProperty: 'someValue',
    //对象字面量可以包含属性和方法
    //例如：可以声明模块的配置对象
    myConfig: {
        useCaching: true,
        language:'zh'
    },
    //基本方法
    myMothed: function() {
        console.log('hello world!');
    },
    //根据当前配置输出信息
    myMothed2: function() {
        console.log('Caching is:' + (this.myConfig.useCaching) ? 'enabled' : 'dissabled');
    },
    //重写当前的配置
    myMothed3: function(newConfig) {
        if(typeof newConfig === 'object') {
            this.myConfig = newConfig;
            console.log(this.myConfig.language);
        }
    }
};
//使用
console.log(myModule.myMothed());//hello wrold
console.log(myModule.myMothed2());//enabled;
console.log(myModule.myMothed3({language:'en',useCaching: false})); //en

使用对象字面量有利于封装和组织代码

  • Module(模块)模式
    能够使一个单独的对象拥有公有/私有的属性和方法，从而屏蔽来自全局作用域的特殊部分，产生的结果是：函数名与在页面上其它脚本定义的函数冲突的可能性降低；Module模式使用闭包封装‘私有’状态和组织，防止其泄露至全局作用域，并与别的开发者的接口发生冲突。通过该模式，只需返回一个公有API，其他的一切则都维持在私有闭包里。该模式除了返回一个对象而不是一个函数外，非常类似一个立即调用的函数表达式。

一个自包含的Module：

var myModule = (function() {
    var counter = 0;
    return {
        incrementCounter:function() {
            return ++counter;
        },
        resetCounter:function() {
            console.log('counter value prior to reset:' + counter);
            counter = 0;
        }
    }
    })();
    console.log(myModule.incrementCounter());

counter变量只能通过返回的两个函数来操作;
一个包含命名空间、公有、私有变量的Module：

var myNamespace = (function() {
    //私有计数器变量
    var myPrivateVar = 0;
    //记录所有参数的私有函数
    var myPrivateMethod = function(foo) {
        console.log(foo);
    };
    return {
        //公有变量
        myPublicVar: "foo",
        //调用私有变量和方法的公有函数
        myPublicFunction: function(bar) {
            //增加私有计数器的值
            myPrivateVar++;
            //传入bar调用私有方法
            myPrivateMethod(bar);
        }
    };
    })();
    console.log(myNamespace.myPublicVar);//foo
    console.log(myNamespace.myPublicFunction('hello world!'));//hello world!
  • Module模式变化
    • 引入混入
      模式的这种变化演示了全局变量（如：jQuery）如何作为参数传递给模块的匿名函数。并允许我们按照我们所希望的为他们取个本地别名；

var moModule = (function(jQ,_) {

  • 引出
    允许我们声明全局变量而不需要实现他们，并且同样地支持上一个实例中的全局引入的概念；

代码：

var myModule = (function() {
    //模块对象
    var module = {},
        privateVar = 'hello world1';
        function privateMethod() {
            console.log('foo');
        }
        module.publicProperty = 'foobar';
        module.publicMethod = function() {
            console.log(privateVar);
        };
        return module;
    })();
    console.log(myModule.publicProperty);//foobar
  • Revealing Module(揭示模块)模式
    Module模式的改进版，能够在私有范围内简单定义所有的函数和变量，并返回一个匿名对象，它拥有指向私有函数的指针

代码：

var myRevealingModule = function() {
    var privateVar = 'Ben Cherry',
        publicVar = "Hey there";
    function privateFunction() {
        console.log('Name:'+ privateVar);
    }
    function privateSetName(strName) {
        privateVar = strName;
    }
    function publicGetName() {
        privateFunction();
    }
    //将暴露的公有指针指向到私有函数和属性上
    return {
        setName: publicSetName,
        greeting: publicVar,
        getName: publicGetName
    };
}
  • Singleton(单例)模式
    类的实例化次数只能是一次

代码：

var mySingleton = (function() {
    //实例保持了Singleton的一个引用
    var instance;
    function init() {
        //singleton
        //私有方法和变量
        function privateMethod() {
            console.log('i am private');
        }
        var privateVar = 'i am also private';
        var privateRandomNum = Math.random();
        return {
            publicMethod: function() {
                console.log('the public can see me!');
            },
            publicProperty: 'i am also public',
            getRandomNum:function() {
                return privateRandomNum;
            }
        };
    }
    return {
        //获取singleton的实例，如果存在就返回，不存在就创建新实例
        getInstance: function() {
            if(!instance) {
                instance = init();
            }
            return instance;
        }
    };
    })();
    var singleA = mySingleton.getInstance();
    var singleB = mySingleton.getInstance();
    console.log(singleA === singleB);

新增属性

autocomplete

规定输入字段是否应该启用自动完成功能.值为off或者是on，默认是开启。自动完成允许浏览器预测对字段的输入。当用户在字段开始键入时，浏览器基于之前键入过的值，应该显示出在字段中填写的选项,就是当你提交表单后，输入框会保留你之前输入的东西,下次输入时，会开始是一样的，会提示你之前输入的值
autocomplete 属性适用于 <form>，以及下面的 <input> 类型：text, search, url, telephone, email, password, datepickers, range 以及 color

autofocus

规定当页面加载时 input 元素应该自动获得焦点,也就是加载完之后，就处于选中状态，不用用鼠标来来获得选中状态

form属性

规定位于表单之外的输入字段，应该属于哪个表单
form 属性规定 input 元素所属的一个或多个表单。
form 属性的值必须是其所属表单的id。
如需引用一个以上的表单，请使用空格分隔的列表

formaction

formaction 属性 覆盖 form 元素的 action 属性。
该属性适用于 type=”submit” 以及 type=”image”

formenctype

formenctype 属性 覆盖 form 元素的 enctype 属性。
该属性与 type=”submit” 和 type=”image” 配合使用

formaction

formmethod 属性覆盖 form 元素的 method 属性
该属性与 type=”submit” 以及 type=”image” 配合使用

formmethod

formmethod 属性覆盖 form 元素的 method 属性
该属性与 type=”submit” 以及 type=”image” 配合使用

formnovalidate

formnovalidate 属性覆盖 form 元素的 novalidate 属性。
如果使用该属性，则提交表单时按钮不会执行验证过程。
该属性适用于 <form> 以及以下类型的 <input>：text, search, url, telephone, email, password, date pickers, range 以及 color,如果设置了该属性，这些类型都不会进行验证了，例如，type="email",如果没有设置这个属性，那么要验证Email是否合乎email的格式，如果设置了就不进行验证了

formtarget

formtarget 属性覆盖 form 元素的 target 属性。
该属性与 type=”submit” 以及 type=”image” 配合使用

height，width

height，width 属性只适用于 <input type="image">，它规定 image input 的高度或宽度。
提示：为图片指定高度和宽度是一个好习惯。如果设置了这些属性，当页面加载时会为图片预留需要的空间。而如果没有这些属性，则浏览器就无法了解图像的尺寸，也就无法为其预留合适的空间。情况是当页面和图片加载时，页面布局会发生变化.值可以是像素或者是百分数

list

list 属性引用数据列表，其中包含输入字段的预定义选项

<form action="demo_form.asp">
Webpage: <input type="url" (list="url_list") name="link" />
<datalist (id="url_list")>
<option label="W3Schools" value="http://www.w3schools.com" />
<option label="Google" value="http://www.google.com" />
<option label="Microsoft" value="http://www.microsoft.com" />
</datalist>

input输入框会有一个下拉式菜单

min,max

带有指定数字范围的，min指定最小值，max指定最大值
max 属性与 min 属性配合使用，可创建合法值范围。
max 和 min 属性适用于以下 <input> 类型：number, range, date, datetime, datetime-local, month, time 以及 week。

number 数字值。规定输入字段允许的最大值。
date 日期。规定输入字段允许的最大值

pattern

pattern 属性规定用于验证输入字段的模式
这个模式是一个正则表达式字符串，pattern 属性适用于以下 类型：text, search, url, telephone, email 以及 password 。
最好配合title属性来使用提示你，你使用的模式

placeholder

用于提示那个输入框应该输入什么。当输入框为空的时候，显示，当输入框不为零时，自动消失
placeholder 属性适用于以下的 类型：text, search, url, telephone, email 以及 password。

readonly

输入框的里的文字为只读的，不能进行修改，不过，用户仍然可以使用 tab 键切换到该字段，还可以选中或拷贝其文本。可以使用js来取消。readonly 属性可与 <input type="text"> 或 <input type="password"> 配合使用

required

有这个值代表这个输入框必须有值，不能为空。
required 属性适用于以下<input>类型：text, search, url, telephone, email, password, date pickers, number, checkbox, radio 以及 file

step

规定数字输入的间隔，例如：step=3，数字的增加就以3为间隔。value规定初值，适用于以下 <input> 类型：number, range, date, datetime, datetime-local, month, time 以及 week。

type属性

type规定了input元素的类型

<input type="text"/>
这个是最常见的输入文本类型，这个只能输入单行文本，通过获取元素的value来获取其中的值
<input type="button"/>
这个定义了一个按钮，但是没有任何行为，就是一个单独的按钮，这和submit不同。你可以为它指定点击事件
<input type="checkbox/>"
定义一个复选框，你可以多选，选择一个或者是多个，用name属性来指定这是不是同一类复选框，这个属性一定要有的。value属性，来指定input元素的值
<input type="file"/>
用于文件上传
<input type="hidden"/>
定义一个隐藏的字段，隐藏一个input标签，可以通过js获取到
<input type="image"/>
定义了一个图片式的提交按钮，这个一定要有’src’属性来指定一个图片的URL，最好还有一个alt，当图片不能显示时，显示的内容
<input type="password"/>
这个是用于输入密码的，输入的密码会被星号或者是点号隐藏
<input type="radio"/>
定义一个单选框，你可以定义一定数目的单选框，只能选择一个，用name来指定同一类，value指定每个单选框的值

value属性

不同的type属性，value属性表示的也不同

  • type=”checkbox”, “radio”, “image” - 定义与输入相关联的值，也就是点击那个按钮的值

元素鼠标事件中

offsetX,offsetY

获取的是点击位置距离元素左边界和上边界的距离大小；

clientX,clientY

坐标值是鼠标事件发生时鼠标光标在可视区（视口）的位置（我们看得见页面的地方）

layerX,layerY

这个是很麻烦的，它是获取发生鼠标事件时光标距离最近position设置为absolute或者relative元素的距离（本身，或者是祖先元素）,positioin:fixed时很诡异，父元素设置了position,对其是由影响的

pageX,pageY

获取的是鼠标事件发生时鼠标光标相对于整个文档元素（document）的坐标（包含滚动的距离），在页面没有滚动时，值和clientX，clineY是一样的

screenX,screenY

获取的鼠标事件发生时鼠标光标相对于电脑屏幕的坐标

x,y

这个值始终和clientX，clinetY，一样

元素的大小

偏移量

offsetWidth,offsetHeight

这个是获取的元素的可视大小，包括padding（内边距），border（边距），content（内容区），如果有滚动条还包含滚动条

offsetLeft,offsetTop

获取跟 直接 父元素的偏移量,从元素的border到父元素的content区域,offsetParent,是获取的包含它的父元素

客户区的大小

clientWidth,clientHeight

只包含padding和content（内容区），千万注意，不包括border，也不包括滚动条的宽度
ps.偏移量和客户区大小都是只读的

滚动大小

scrollWidht,scrollHeight

获取包含元素的大小，在没有滚动条时，和clientWidth，clientHeight是一样的.在有滚动条的时候，只包含content和padding-left，padding-top，但是chrome还有padding-bottom

scrollTop,scrollLeft

如果元素没有滚动条它们的值就是0；
获取滚动条滚动的距离，可以对其进行赋值，使滚动条到指定的位置

PS,特殊元素下获取:document.documentElement,document.body

getBoundingClientRect()

它不需要参数，返回一个有left,right,top,bottom，width，height的对象,其中width和height在IE9+上支持，获取的width，height跟offsetWidhth，offsetHeight等价

getClientsRects()

这个是可以获取一个内联元素（如span）多行文本中的 每一行 的位置信息

window

window.innerHeight,window.innerWidth

获取浏览器可视显示文档的地方的大小,有滚动条则包括滚动条

window.outerHeight,window.outerWidth

获取的是浏览器窗口的大小
图片中的红框就是inner，整个窗口就是outer
在全屏的时候inner和outer始终是一样的

window.pageXOffset和window.pageYOffset,window.scrollX和window.scrollY

他们是对应等价的，都是获取滚动条滚动的高度
跨浏览器获取文档的滚动距离

var x = (window.pageXOffset !== undefined) ? window.pageXOffset : (document.documentElement || document.body.parentNode || document.body).scrollLeft;
var y = (window.pageYOffset !== undefined) ? window.pageYOffset : (document.documentElement || document.body.parentNode || document.body).scrollTop;

window.screenX和window.screenY,window.screenLeft和window.screenTop

他们是对应相等的，都是获取的浏览器窗口相对于电脑屏幕的偏移的距离

CSS之line-height

發表於 2015-06-18   |   分類於 css

今天看了张鑫旭大神关于line-height属性的视频我就记录一下我的观看笔记

line-height

定义

两行文字基线之间的距离
什么是基线？
就像我们四线英语本中的第三根线
为什么是基线？
其它的线都是以基线为依据

  • 不同字体，对基线参考稍微不同，有的紧挨，有的不紧挨
  • line-height可以让单行内联元素垂直居中，注意font-size要设置为0

line-height与行内框盒子模型

<p>张大婶<em>大神</em>heihei</p>
这一行包含四个盒子

  • 内容区域（content area）,是一种围绕文字看不见的盒子，大小和font-size大小相关
  • 内联盒子（inline boxes），不会让内容成块显示，而是排成一行，如果外部含inline水平标签（span，a，em等），则属于内联盒子。如果是个光秃秃的文字，则属于匿名内联盒子；例如：上面的‘张大婶’，‘heihei’就是匿名内联盒子，而’大神’就是内联盒子
  • 行框盒子（line boxes），每一行就是一个行框盒子，每个行框盒子又是由一个个内联盒子组成
  • p标签所在的‘包含盒子’（containing box），此盒子是由一行行的行框盒子组成

line-height高度机理

  • 内联元素的高度是line-height决定的，跟font-size一点关系也没有
  • 高度的表现不是行高，行高只是幕后黑手，真正决定高度是内容区域和行间距，只不过内容区域高度 + 行间距 == 行高
  • 内容区域的高度只与字号（font-size）和字体(font-family)有关，与line-height没有关系

line-height属性值

  • normal — 跟着用户的浏览器有关，且与元素字体关联
  • number — 根据当前元素的font-size大小计算
  • length
  • percent — 相对于设置了该line-height属性的元素的font-size大小计算
  • inherit

行高与图片表现

消除图片底部间隙的方法

  1. 图片块状化 - 无基线对齐
    img{display:block;}
  2. 图片底线对齐
    img{vertical-align:bottom;}
  3. 行高足够小 - 基线位置上移
    .box{line-height:0;}

line-height实际应用

  • 实现大小不固定的图片、多行文字垂直居中
    • 图片水平垂直居中
      .box{line-height:300px;text-align:center;}
      .box > img{vertical-align:middle;}
    • 多行文本水平垂直居中
      .box {line-height:250px;text-align:center;}
      .box > .text{display:inline-block; line-height:normal;text-align:left:vertical-align:middle;}
