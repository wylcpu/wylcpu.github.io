iOS之viewController生命周期

发表于 2016-01-11   |  

与viewController生命周期有关的方法

通过了解控制器的生命周期，进而知道应该在什么时候处理自己的代码了；

+(void)initialize;//类的初始化
-(id)init;//对象的初始化
-(id)initWithCode:(NSCoder*)aDecoder;//归档的初始化
-(void)loadView;//加载视图
-(void)viewDidLoad将要加载视图
-(void)viewWillAppear;//视图将要显示
-(void)viewWillLayoutSubviews;//将要布局子视图
-(void)viewDidLayoutSbubiews;//完成布局子视图
-(void)viewDidAppear:(BOOL)animated;//视图显示
- (void)viewWillDisappear:(BOOL)animated//视图将要消失
- (void)viewDidDisappear:(BOOL)animated//视图已经消失了
-(void)dealloc//对象被释放

例子

+ (void)initialize{
    [super initialize];
    NSLog(@"你是initialize");
}

-(id)initWithCoder:(NSCoder *)aDecoder{
    self = [super initWithCoder:aDecoder];
    NSLog(@"你是initWithCoder");
    return self;
}
-(id)init{
    self = [super init];
    NSLog(@"你是init");
    return self;
}
- (void)viewDidLoad {
    [super viewDidLoad];
    // Do any additional setup after loading the view.
    NSLog(@"你是viewDidLoad");
}

- (void)didReceiveMemoryWarning {
    [super didReceiveMemoryWarning];
    // Dispose of any resources that can be recreated.
}
-(void)loadView{
    [super loadView];
    NSLog(@"你是loadView");
}
- (void)viewWillAppear:(BOOL)animated{
    [super viewWillAppear:animated];
    NSLog(@"你是viewWillAppear");
}
- (void)viewDidAppear:(BOOL)animated {
    [super viewDidAppear:animated];
    NSLog(@"你是viewDidAppear");
}
- (void)viewWillLayoutSubviews {
    [super viewWillLayoutSubviews];
    NSLog(@"你是viewWillLayoutSubviews");
}
- (void)viewDidLayoutSubviews {
    [super viewDidLayoutSubviews];
    NSLog(@"你是viewDidLayoutSubviews");
}
- (void)viewWillDisappear:(BOOL)animated {
    [super viewWillDisappear:animated];
    NSLog(@"你是viewWillDisappear");
}
- (void)viewDidDisappear:(BOOL)animated {
    [super viewDidDisappear:animated];
    NSLog(@"你是viewDidDisappear");
}

-(void)dealloc {
    NSLog(@"你是dealloc");
}

总结

  1. initialize：只有在这个类第一次创建对象时才会调用。以后创建对象也都不会再调用。因此，如果我们有一些和这个相关的全局变量，可以在这里进行初始化
  2. init：代码初始化调用
  3. initWithCoder：使用nib或storyboard的时候调用
  4. viewDidLoad方法是我们最常用的方法的，类中成员对象和变量的初始化我们都会放在这个方法中
  5. loadView方法是开始加载视图的起始方法
  6. viewWillAppear：视图将要展现时会调用。
  7. viewWillLayoutSubviews：在viewWillAppear后调用，将要对子视图进行布局。
  8. viewDidLayoutSubviews：已经布局完成子视图。
  9. viewDidAppare：视图完成显示时调用。
  10. viewWillDisappear：视图将要消失时调用。
  11. viewDidDisappear：视图已经消失时调用。
  12. dealloc：controller被释放时调用。
    其中，viewWillAppear,viewDidAppear,viewWillLayoutSubviews,viewDidLayoutSubviews,viewWillDisappear,viewDidDisppear是在push和pop的时候都调用的。

关于java的流程控制，由于跟其他语言类似，我就列举容易出现问题的地方

switch

  1. switch 后面小括号中表达式的值必须是整型或字符型
  2. case 后面的值可以是常量数值，如 1、2；也可以是一个常量表达式，如 2+2 ；但不能是变量或带有变量的表达式，如 a * 2
  3. case 匹配后，执行匹配块里的程序代码，如果没有遇见 break 会继续执行下一个的 case 块的内容，直到遇到 break 语句或者 switch 语句块结束

for执行过程

  1. 执行循环变量初始化部分，设置循环的初始状态，此部分在整个循环中只执行一次
  2. 进行循环条件的判断，如果条件为 true ，则执行循环体内代码；如果为 false ，则直接退出循环
  3. 执行循环变量变化部分，改变循环变量的值，以便进行下一次条件判断
  4. 依次重新执行< 2 >、< 3 >、< 4 >，直到退出循环
