xss入门

什么是xss

  • 非法转账
  • 挂马

nodejs学习笔记一

发表于 2015-06-01   |   分类于 nodejs   |  
  1. nodejs是一个异步事件驱动的平台

process

  • process是一个全局内置对象，可以在代码中德任何位置访问此属性，这个对象代表我们的nodejs代码宿主的操作系统进程对象。
  • process.cwd();查看应用程序的当前目录
  • process.chdir("dirpath")，改变应用程序的目录
  • stdout是标准输出流，console.log = function(d) {process.stdout.write(d+'\n')};console.log就是封装的它；
  • stderr是标准错误流，我们可以通过它来捕获错误信息process.stderr.write("error")
  • stdin是进程的输入流
  • exit是在程序内杀死进程

文件I/O

  • nodejs中提供了一个名为fs的模块来支持I/O操作
  • writeFile函数可以异步的将数据写入一个文件，如果文件已经存在则会被替换;writeFile(filename,data,callback);回掉函数只有一个err参数
  • appendFile函数和writeFile差不多，区别是：它可以将新的内容追加到已有的文件中，如果文件不存在就创建一个新文件。
  • exists函数检查一个文件是否存在exists(file,clallback);回调函数只有一个蚕食，类型是布尔值，true表示存在文件，false表示没有这个文件
  • rename函数：修改文件名rename(oldFileName,newFileName,callback(err));也可以用来移动文件rename(oldPath,newPath,callback(err));
  • readFile(file,encode,callback(err,data));回掉中的data就是读取的数据
  • unlink(file,callback(err))函数，删除一个文件
  • mkdir(path[,权限],callback(err))函数创建一个目录,
  • rmdir(path,callback(err));删除目录
  • readdir(目录,callback(err,files)),回调函数中的files是文件名数组

URL处理

  • 首先引入模块：var url = require('url')
  • parse函数是解析url，返回一个json格式的数组

example:

url.parse('http://www.evil007.com:8888/blog/mm?id=1&age=22')
//打印：
{
protocol:'http:'
slashes: true,
auth: null,
host: 'www.evil007.com:8888',
port: '8888',
hostname: 'www.evil007.com',
hash: null,
search: '?id=1&age=22',
query: 'id=1&age=22',
pathname: '/blog/mm',
path: '/blog/mm?id=1&age=22',
href: 'http://www.evil007.com:8888/blog/mm?id=1&age=22'
 }

如果第二个参数为true，表示条件解析，查询条件也会解析成json格式对象
query: { id: '1', age: '22' },,其他同上
如果有第三个参数为true，表示解析时会将url中//和第一个/之间的部分解析为主机名

  • format(json)函数的作用与parse相反，它的参数是一个json对象，返回一个组装好的url地址
  • resolve函数的参数是两个路径，第一个路径是开始的路径或者说当前路径，第二个则是想要去往的路径，返回值是一个组装好的url

通过指定部分参数，定制新的函数；

MongoDB基本操作

发表于 2015-05-31   |   分类于 MongoDB   |  

看慕课网关于MongoDB介绍，读书笔记

  1. use命令：切换和创建数据库。例：use imooc
  2. show dbs：查看数据库
  3. db.dropDatabase():删除数据库
  4. 创建集合（即表）:db.集合名.insert({});例：db.imooc_collect.insert({x:1});
  5. show collections:查看创建的集合（表）==show tables
  6. 查询数据：db.集合名.find();例：db.imooc_collect.find(),默认显示所有数据
  7. 插入多条数据：for(i=id,i<num;i++)db.imooc_collect.insert({x:i});
  8. db.集合名.find().count()：显示数据数量
  9. db.集合名.find().skip(3).limit(2).sort({x:1})
  10. db.imooc_collect.find({x:5}),查找x=5的项

数据更新和删除

  1. db.imooc_collect.update({x:5});参1：待更新数据，参2：要更新的数据，之后就把x等于5，替换成x等于999 ,默认只是更新查到的第一条数据
  2. db.imooc_collect.insert({x:100,y:100,z:100})
    12.更新：db.imooc_collect.update({z:100},{y:99}),会覆盖掉x，z
    13.部分更新db.iooc_collect.update({z:100},{$set:{y:99}}),结果是：{x:100,y:99,z:100}
    14.更新不存在的数据，db.imooc_collect.update({z:101},{z:333},true);第三个参数就是说如果不存在要替换的数据就创建一条，
  3. 更新多条数据db.imooc_collect.update({z:1},{$set:{z:3}},false,true);
  4. 删除数据：跟查询差不多，只接受一个参数，不允许不传参，db.imooc_collect.remove({x:5});
  5. 删除某张表用db.imooc_collect.drop();

    创建索引

  6. 查看索引：db.imooc_collect.getIndexes();
  7. 创建索引；db.imooc_collect.ensureIndex({x:1}),1为正向排序，-1为逆向排序
  8. _id索引是大多数集合默认建立的索引，对于每个插入的数据，MongoDB都会自动的生成一条唯一的_id字段
    21.单键索引是最普通的索引，单键索引时不会自动创建的,用db.imooc_collect.ensureIndex({x:1});
    22.多键索引与单键索引创建形式相同，区别在于字段的值，单键索引：值为一个单一的值，例如字符串，数字或日期。多键索引：值具有多个记录，例如数组
    23.复合索引db.imooc2.ensureIndex({x:1,y:1});
  9. 过期索引：是在一段时间后会过期的索引，在索引过期后，相应的数据也会被删除的。储存在过期索引字段的值必须是指定的时间类型，说明：必须是ISODate或ISODate数组，不能使时间戳，否则不能被自动删除;如果是指定了ISODate数组，则按照最小的时间进行删除。过期索引不能是复合索引。删除时间是不精确的。
  10. 全文索引：对字符串与字符串数组创建全文可搜索的索引例如：db.imooc_collect.ensureIndex({key:”text”}),db.imooc_collect.ensureIndex({“$**”:”text”});一个数据集合只能创建一个全文索引
  11. 如何使用全文索引进行查询：db.imooc_collect.find({$text:{$search:”content”}}),content中的内容用空格分隔是或查询，例：db.imooc_collect.find({$text:{$search:”aa bb”}}),这就是查询包含aa，或者包含bb的字符串的记录
  12. 全文索引相似度使用$meta操作符：{sore：{$meta:”textScore”}}写在查询条件后面可以返回结果的相似度，与sort一起使用，效果更好，例：db.imooc_collect.find({$text:{$search:”aa bb”}}，{score:{$meta:”textScore”}});如果与sort一起使用db.imooc_collect.find({$text:{$search:”aa bb”}}，{score:{$meta:”textScore”}}).sort({score:{$meta:”textScore”}});
    28.全文索引的使用限制，每次查询只能指定一个$text查询；$text查询不能出现在$nor查询中；查询中如果包含$text,hint不再起作用；现在mongodb还不支持中文的全文索引
  13. 创建索引时的格式：db.collection.ensureIndex({param},{param}),其中，第二个参数便是索引的属性，其中比较重要的属性有：名字，name指定：db.collection.ensureIndex({},{name:””}),默认的名字是用下划线的;唯一性，unique指定：db.collection.ensureIndex({},{unique:true/false});稀疏性，sparse指定：db.collection.ensureIndex({},{spare:true/false});过期索引；
    30.地理位置索引：将一些点的位置存储在MongoDB中，创建索引后，可以按照位置来查找其他点。子分类：2d索引，用于存储和查找平面上的点；2dsphere索引，用于存储和查找球面上的点；查找方式：1.查找距离某个点一定距离的点。2.查找包含在某区域内的点。2D索引：创建方式：db.collection.ensureIndex({w:”2d”}),位置表示方式：经纬度[经度][维度]，取值范围：经[-180，180]，纬[-90][90]，例，db.location.insert({w:[45,45]}),查询方式：$near查询：查询距离某个点最近的点,例：db.location.find({w:{$near:[25][25]}}),db.location.find({w:{$near:[25,25],$maxDistance:10}})。$geoWithin查询：查询某个形状内的点,形状的表示：1.$box:矩形，使用{$box:[[,],[,]]}表示，例如：db.locatioin.find({w:{$geoWithin:{$box:[[x1,y1],[x2,y2]]}}})2.$center:圆形，使用{$center:[[x1,y1],r]},3.$polygon:多边形，使用{$polygon:[[x1,y1],[x2,y2],[x3,y3]]};
    31.geoNear查询：geoNear使用runCommand命令：db.ruCommand({geoNear:,near:[x,y],minDistance:(2d索引失效),maxDistance:,num:,—-})
    32.2dsphere索引：创建方式：db.collection.ensureIndex(w:”2dsphere”),位置表示方式：GeoJSON:描述一个点，一条直线，多边形等形状，格式：{type:””,coordinates:[<coordinates]},查询方式和2d索引查询方式类似，支持$minDistance与$maxDistance
    33.索引构建情况分析：索引的好处：加快索引相关的查询。索引的缺点是：增加磁盘空间消耗，降低写入性能。
  14. 如何评判当前索引构建情况：1.mongostat工具介绍2.profile集合的介绍。3.日志的介绍4explain分析
    35.mongostat工具：查看mongodb运行状态的程序
