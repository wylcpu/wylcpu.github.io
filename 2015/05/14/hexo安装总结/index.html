hexo安装总结

发表于 2015-05-14   |   分类于 hexo   |  

终于用hexo搭建blog完成了，虽然说就是简单的几步，要是你搭建过，让你再搭建一个，真是分分钟的事，但是如果你是第一次，并且对相关的东西你不是很熟悉的话，你就有可能遇到很多问题；我就把在此安装中遇到的问题总结一下。

HEXO的安装

  1. 首先你安装了git；
  2. 你也安装了nodeJs；
  3. 你还有github
    我就当你已经安装了git和nodejs

本地运行

打开git bash运行

npm install -g hexo-cli

-g是进行全局安装hexo
之后切换到你想要安装的位置
你也可以先建立好要安装的目录，例如blog
那你之后就cd切换到这个目录下输入下面命令

hexo init
npm install

要是你不提前建立目录就想官方提供似得，我是以建立blog目录为例

hexo init blog
cd blog
npm install

这也是可以的，这个是现在官方上说的
接下来就是

hexo generate

这个命令也可以简写为

hexo g

这个命令是生成静态文件
那就让我们先在本地上运行一下吧，输入

hexo server

在你的浏览器地址栏中输入

localhost:4000

hexo默认是开启的4000端口的
会出现一个hexo默认的主题
那就说明你成功了

部署到github

按官方说，我是用的user site，也就是个人主页了，你也可以使用项目安装，我没
有试过，以后用熟悉了，再各种折腾。

  • 首先你新建一个repository仓库，把名字命名为我就 username.github.io，把斜体替换成你的名字。注意你不能在这个仓库里建立分支，就是默认的就行了。

  • 接下来就是进行配置了，在你的博客目录下找到_config.yml,找的最下面

    deploy:

    type: git
repo: https://github.com/username/username.github.io.git
branch: master

    注意:名值对中的冒号后是有个空格的，我就是没有这个空格，弄了我一晚上各种的折腾
    就是没有用，虽然刚开始万能的Google没有帮我检索出我要的答案还误导我乱弄了半晚上，但是最后我修改了一下关键字就行了。终于让我找到了，真是众里寻他千百度，蓦然回首，她却在这里。我发现他们的官方的github里面的有各种的问题解决方法，我敢保证，你遇到的问题一定在这里有hexo官方问题集.

1.接下来安装官方说你还有执行一条命令，我刚开始没有使用就不对，

npm install hexo-deployer-git -save

2.接下来你在运行

hexo generate
hexo d

到这里是很容易出错的，很多人在这个地方是出各种的错误，有错误就先看我上面说的官方的地方找找。
接下来输入用户名和密码就行了
如果能成功运行那么你就可以在地址栏输入

username.github.io

这就出现了你在本地成功测试时出现的界面

总结

其实就几条命令很简单，如果没有出现任何问题，我保证你十分钟，弄完，但是如果你初次接触，你就需要看各种文档，你才明白，这些命令时干什么的，并且有可能一个问题
让你弄很长时间，最终你花费了N hours弄好了。但是出现问题才使你成长，要是你十分钟弄完，你什么也学不到，要的就是这种不顺利。
