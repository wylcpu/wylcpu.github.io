CSS之line-height

发表于 2015-06-18   |   分类于 css   |  

今天看了张鑫旭大神关于line-height属性的视频我就记录一下我的观看笔记

line-height

定义

两行文字基线之间的距离
什么是基线？
就像我们四线英语本中的第三根线
为什么是基线？
其它的线都是以基线为依据

  • 不同字体，对基线参考稍微不同，有的紧挨，有的不紧挨
  • line-height可以让单行内联元素垂直居中，注意font-size要设置为0

line-height与行内框盒子模型

<p>张大婶<em>大神</em>heihei</p>
这一行包含四个盒子

  • 内容区域（content area）,是一种围绕文字看不见的盒子，大小和font-size大小相关
  • 内联盒子（inline boxes），不会让内容成块显示，而是排成一行，如果外部含inline水平标签（span，a，em等），则属于内联盒子。如果是个光秃秃的文字，则属于匿名内联盒子；例如：上面的‘张大婶’，‘heihei’就是匿名内联盒子，而’大神’就是内联盒子
  • 行框盒子（line boxes），每一行就是一个行框盒子，每个行框盒子又是由一个个内联盒子组成
  • p标签所在的‘包含盒子’（containing box），此盒子是由一行行的行框盒子组成

line-height高度机理

  • 内联元素的高度是line-height决定的，跟font-size一点关系也没有
  • 高度的表现不是行高，行高只是幕后黑手，真正决定高度是内容区域和行间距，只不过内容区域高度 + 行间距 == 行高
  • 内容区域的高度只与字号（font-size）和字体(font-family)有关，与line-height没有关系

line-height属性值

  • normal — 跟着用户的浏览器有关，且与元素字体关联
  • number — 根据当前元素的font-size大小计算
  • length
  • percent — 相对于设置了该line-height属性的元素的font-size大小计算
  • inherit

行高与图片表现

消除图片底部间隙的方法

  1. 图片块状化 - 无基线对齐
    img{display:block;}
  2. 图片底线对齐
    img{vertical-align:bottom;}
  3. 行高足够小 - 基线位置上移
    .box{line-height:0;}

line-height实际应用

  • 实现大小不固定的图片、多行文字垂直居中
    • 图片水平垂直居中
      .box{line-height:300px;text-align:center;}
      .box > img{vertical-align:middle;}
    • 多行文本水平垂直居中
      .box {line-height:250px;text-align:center;}
      .box > .text{display:inline-block; line-height:normal;text-align:left:vertical-align:middle;}
